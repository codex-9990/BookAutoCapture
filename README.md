# BookAutoCapture

BookAutoCaptureは、固定したAndroid端末で本のページを連続撮影するためのCameraXアプリです。本を机に置き、ページを手でめくった後に画面が安定すると、高画質JPEGを自動で1枚保存します。

OCR、PDF化、AI連携、クラウド送信、広告、ログイン、課金、外部解析SDKは実装していません。画像は端末内のMediaStoreにのみ保存します。

## 主な機能

- CameraX Previewによるカメラプレビュー
- CameraX ImageCaptureによるJPEG保存
- CameraX ImageAnalysisによる軽量な輝度差分解析
- 明るさ、ブレ、安定状態のライブ品質チェック
- ページめくり検出後、安定待ち時間を満たしたら自動撮影
- 撮影状態、撮影済み枚数、次に保存されるファイル名の表示
- 大きな `撮影を開始` / `中断する` / `続きから撮る` ボタンを中心にしたシンプルな操作パネル
- アプリ終了後も同じセッションを続けられる中断/再開
- 横長写真をデフォルトにした写真向き選択
- 写真の向きに合わせた横長/縦長プレビュー枠
- セッション内で保存したページ名の簡易リスト表示と復元
- 確認付きの最後に保存したページの削除
- 確認付きの新しい本の開始
- 撮影完了音のON/OFF
- 安定待ち時間、最短撮影間隔、感度、ブレ判定、暗さ判定の設定
- 横向き撮影用の左右分割レイアウト
- 自動撮影中の画面スリープ抑止
- 起動時の `cacheDir/tmp` クリーンアップ

## ビルド方法

Android Studioでこのフォルダを開き、Gradle同期後に `app` をビルドします。

コマンドラインでビルドする場合:

```powershell
.\gradlew.bat assembleDebug
```

このプロジェクトは以下を前提にしています。

- minSdk 29
- targetSdk 36
- compileSdk 36
- Kotlin
- Jetpack Compose
- CameraX

## 実機での起動方法

1. Android Studioで端末を選択します。
2. `app` を実行します。
3. 初回起動時にカメラ権限を許可します。
4. 本全体がプレビューに入るように端末を固定します。
5. `撮影を開始` を押して、ページをめくります。
6. 画面が安定すると自動でJPEGが保存されます。

横向きにすると、プレビューを左側、操作パネルを右側に表示します。保存写真は見開き本に合わせて横長がデフォルトです。操作パネルの `写真の向き` から縦長にも切り替えられます。自動撮影中は画面が暗くなったりスリープしたりしないようにします。

## 中断と再開

撮影中に `中断` を押すと、現在のセッション、保存先、撮影済みページ一覧を端末内に保存します。

アプリを閉じた後でも、次回起動時に `続きから撮る` を押すと同じフォルダの次の連番から撮影できます。別の本を始める場合は `新しい本を始める` を使います。

## 保存先

撮影画像はAndroid 10以上のMediaStoreへ保存します。ストレージ権限は要求しません。

保存先はセッション開始時刻ごとのフォルダです。

```text
Pictures/BookAutoCapture/YYYYMMdd_HHmmss/page_0001.jpg
Pictures/BookAutoCapture/YYYYMMdd_HHmmss/page_0002.jpg
Pictures/BookAutoCapture/YYYYMMdd_HHmmss/page_0003.jpg
```

ファイル名は `page_0001.jpg` から連番です。

## 撮影音OFFの制限

`撮影音` をOFFにすると、アプリ側の撮影完了音は鳴りません。

ただし、Android端末、地域、メーカー仕様により、OSまたはカメラサービス側のシャッター音が強制される場合があります。このアプリはroot、ADB、システム設定改変、非公開APIによる回避は行いません。

## 推奨撮影環境

- スマホを固定する
- 本全体が画面に入る位置にする
- 明るい照明を使う
- ページをめくった後、0.5〜1秒だけ手を止める
- 指が写ってもよいが、文字を隠しすぎない

## 実機テスト手順

1. プレビューが表示されることを確認します。
2. `1枚だけ撮る` で1枚保存できることを確認します。
3. 品質チェックが `良好`、`注意`、`撮影しません` のいずれかで表示されることを確認します。
4. `写真の向き` がデフォルトで `横長` になっていることを確認します。
5. `写真の向き` を切り替え、プレビュー枠と保存画像の向きが横長/縦長で変わることを確認します。
6. `撮影を開始` を押して状態が `ページめくり待ち` になることを確認します。
7. ページをめくり、状態が `ページめくり検出` から `安定待ち` に変わることを確認します。
8. 画面を約1秒止め、自動で1枚保存されることを確認します。
9. `Pictures/BookAutoCapture/YYYYMMdd_HHmmss/` に連番JPEGが保存されることを確認します。
10. 保存リストにファイル名が表示されることを確認します。
11. 自動撮影を `中断` してから `最後を削除` を押し、確認後に直近の画像が削除されることを確認します。
12. `撮影音` をOFFにして、アプリ側の撮影完了音が鳴らないことを確認します。
13. 自動撮影中に端末がスリープせず、画面が暗くならないことを確認します。
14. 端末を横向きにして、プレビューと操作パネルが左右に並ぶことを確認します。
15. 横向きのまま手動撮影または自動撮影し、保存画像の向きが自然であることを確認します。
16. アプリを閉じて起動し直し、`続きから撮る` で同じ保存先と次の連番から続けられることを確認します。
17. `新しい本を始める` で確認後に新しい保存先の `page_0001.jpg` から始まることを確認します。
18. 暗い場所や大きくブレた状態で、品質チェックが `撮影しません` になり、必要に応じて `暗すぎます` または `ブレています` が表示されることを確認します。

## ユニットテスト

自動撮影の状態機械は端末なしでテストできます。

```powershell
.\gradlew.bat testDebugUnitTest
```

テスト対象:

- ページめくり検出後、安定時間を満たすと撮影要求が出る
- 安定時間が足りない場合は撮影しない
- クールダウン中は撮影しない
- 暗すぎる場合は撮影しない
- ブレ判定NGの場合は撮影しない
- 品質チェックが良好、注意、撮影不可を判定する

## 既知の制限

- OCR、PDF化、台形補正、曲面補正、ページ分割は未実装です。
- 品質チェック、ブレ判定、暗さ判定は輝度プレーンに基づく簡易判定です。
- 復元される保存リストはアプリが記録したMediaStore URIに基づきます。外部アプリで画像を削除した場合は一覧と実ファイルがずれることがあります。
- 端末仕様によりシステムシャッター音を完全には制御できない場合があります。
- 画角、照明、紙質によってページめくり検出の感度調整が必要になる場合があります。
