# BookAutoCapture

BookAutoCaptureは、固定したAndroid端末で本のページを連続撮影するためのCameraXアプリです。本を机に置き、ページを手でめくった後に画面が安定すると、高画質JPEGを自動で1枚保存します。

OCR、PDF化、AI連携、クラウド送信、広告、ログイン、課金、外部解析SDKは実装していません。画像は端末内のMediaStoreにのみ保存します。

## 主な機能

- CameraX Previewによるカメラプレビュー
- CameraX ImageCaptureによるJPEG保存
- CameraX ImageAnalysisによる軽量な輝度差分解析
- ページめくり検出後、安定待ち時間を満たしたら自動撮影
- 撮影枚数、状態、保存先、最後に保存したファイル名の表示
- 大きな開始/停止ボタンを中心にしたシンプルな操作パネル
- セッション内で保存したページ名の簡易リスト表示
- 最後に保存したページの削除
- 撮影完了音のON/OFF
- 安定待ち時間、最短撮影間隔、感度、ブレ判定、暗さ判定の設定
- 横向き撮影用の左右分割レイアウト
- 自動撮影中の画面スリープ抑止
- 起動時の `cacheDir/tmp` クリーンアップ

## ビルド方法

Android Studioでこのフォルダを開き、Gradle同期後に `app` をビルドします。

コマンドラインでビルドする場合:

```powershell
.\gradlew.bat assembleDebug
```

このプロジェクトは以下を前提にしています。

- minSdk 29
- targetSdk 36
- compileSdk 36
- Kotlin
- Jetpack Compose
- CameraX

## 実機での起動方法

1. Android Studioで端末を選択します。
2. `app` を実行します。
3. 初回起動時にカメラ権限を許可します。
4. 本全体がプレビューに入るように端末を固定します。
5. `開始` を押して、ページをめくります。
6. 画面が安定すると自動でJPEGが保存されます。

横向きにすると、プレビューを左側、操作パネルを右側に表示します。自動撮影中は画面が暗くなったりスリープしたりしないようにします。

## 保存先

撮影画像はAndroid 10以上のMediaStoreへ保存します。ストレージ権限は要求しません。

保存先はセッション開始時刻ごとのフォルダです。

```text
Pictures/BookAutoCapture/YYYYMMdd_HHmmss/page_0001.jpg
Pictures/BookAutoCapture/YYYYMMdd_HHmmss/page_0002.jpg
Pictures/BookAutoCapture/YYYYMMdd_HHmmss/page_0003.jpg
```

ファイル名は `page_0001.jpg` から連番です。

## 撮影音OFFの制限

`撮影音` をOFFにすると、アプリ側の撮影完了音は鳴りません。

ただし、Android端末、地域、メーカー仕様により、OSまたはカメラサービス側のシャッター音が強制される場合があります。このアプリはroot、ADB、システム設定改変、非公開APIによる回避は行いません。

## 推奨撮影環境

- スマホを固定する
- 本全体が画面に入る位置にする
- 明るい照明を使う
- ページをめくった後、0.5〜1秒だけ手を止める
- 指が写ってもよいが、文字を隠しすぎない

## 実機テスト手順

1. プレビューが表示されることを確認します。
2. `手動撮影` で1枚保存できることを確認します。
3. `開始` を押して状態が `ページめくり待ち` になることを確認します。
4. ページをめくり、状態が `ページめくり検出` から `安定待ち` に変わることを確認します。
5. 画面を約1秒止め、自動で1枚保存されることを確認します。
6. `Pictures/BookAutoCapture/YYYYMMdd_HHmmss/` に連番JPEGが保存されることを確認します。
7. 保存リストにファイル名が表示されることを確認します。
8. 自動撮影を停止してから `最後を削除` を押し、直近の画像が削除されることを確認します。
9. `撮影音` をOFFにして、アプリ側の撮影完了音が鳴らないことを確認します。
10. 自動撮影中に端末がスリープせず、画面が暗くならないことを確認します。
11. 端末を横向きにして、プレビューと操作パネルが左右に並ぶことを確認します。
12. 横向きのまま手動撮影または自動撮影し、保存画像の向きが自然であることを確認します。
13. 暗い場所や大きくブレた状態で、必要に応じて `暗すぎます` または `ブレています` が表示されることを確認します。

## ユニットテスト

自動撮影の状態機械は端末なしでテストできます。

```powershell
.\gradlew.bat testDebugUnitTest
```

テスト対象:

- ページめくり検出後、安定時間を満たすと撮影要求が出る
- 安定時間が足りない場合は撮影しない
- クールダウン中は撮影しない
- 暗すぎる場合は撮影しない
- ブレ判定NGの場合は撮影しない

## 既知の制限

- OCR、PDF化、台形補正、曲面補正、ページ分割は未実装です。
- ブレ判定と暗さ判定は輝度プレーンに基づく簡易判定です。
- 保存リストと `最後を削除` は現在のアプリ起動中のセッションが対象です。
- 端末仕様によりシステムシャッター音を完全には制御できない場合があります。
- 画角、照明、紙質によってページめくり検出の感度調整が必要になる場合があります。
